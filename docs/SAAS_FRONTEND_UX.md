# SaaS Frontend UX (Draft)

This outlines the UX flow and key screens for the SaaS.

## 1) Onboarding flow

- Create org and invite team
- Select plan or start trial
- Add first project and target
- Verify target ownership (DNS/HTTP)
- Choose first test suite and run

## 2) Core screens

Dashboard
- Active runs, recent results, findings summary
- Trend charts (pass rate, findings by severity)

Projects
- List of projects with risk indicators
- Quick actions (run suite, view findings)

Targets
- Scope, verification status, and authorization
- Safe scan presets and toggles

Runs
- Run detail view with live logs
- Real-time status, progress, and artifacts
- Cancel or pause run

Findings
- Severity filters, triage workflow
- Assignments, comments, and resolution status
- Export to PDF/CSV

Reports
- Generate and share reports
- Executive summary and technical details

Billing
- Plan usage, invoices, payment methods

Admin
- Team and role management
- Audit log access

## 3) Design system

- Glass/futuristic visual language (consistent with current demo)
- High contrast, readable typography
- Clear hierarchy and safe-scan indicators

## 4) Accessibility

- Keyboard navigation
- Reduced-motion support
- WCAG AA color contrast targets
